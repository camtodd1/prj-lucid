# Repository Instructions

- Work efficiently and keep context small.
- Inspect only files relevant to the request; exclude generated, vendor, build, and dependency directories.
- Use targeted searches and read only relevant file sections.
- Do not restate the task or explain internal reasoning.
- Make the smallest focused change; avoid unrelated refactors.
- Run the narrowest useful tests first.
- Do not repeat unchanged command output.
- If blocked, ask one precise question.
- Final response: summarise changed files, tests run, and any blocker in at most 5 bullets.
