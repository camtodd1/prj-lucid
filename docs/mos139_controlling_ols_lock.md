# MOS139 Controlling OLS Compatibility Lock

## Accepted status — 12 July 2026

MOS139 `ols.controlling_lower_envelope` is supported and locked to the accepted
QGIS 4 output contract. This promotion applies only to the current CASA Part
139 MOS 2019 ruleset. It does not promote modernised Annex 14 OFS/OES or the
modernisation comparison.

The lock covers the committed single, intersecting, parallel, and three-runway
stress fixtures. For each fixture it fixes:

- the complete set of controlling `surface_id` values;
- controlling-region count;
- total solved area; and
- an exact digest of every controller/geometry pair.

The accepted values are stored in
`tests/fixtures/ols/mos139_controlling_lock_qgis4_2026-07-12.json` and are
enforced automatically by `tests/run_ols_workflow_regression.py`. Updating that
file is an explicit compatibility change requiring user approval; it must not
be refreshed simply to make a failing test pass.

The lock was superseded once on 12 July 2026 to correct duplicated ownership at
MOS139 Conical intersections. All Conical/axis merged overlaps are now resolved
from one lower-envelope result and its complement. The corrected four-fixture
matrix has zero cross-controller overlap; YBBN specifically resolves two
TOCS/Conical strips totalling 7.176229 m². The regression runner now treats any
controlling-region overlap above 0.001 m² as a hard failure.

## Supported families and retained behaviour

The locked envelope includes Approach, TOCS, IHS, OHS, Conical, Transitional,
and the existing applicable inner/OFZ surfaces generated by the MOS139 path.
The established same-controller merge normalization is part of the accepted
contract. Existing diagnostics remain available, but Annex-specific production
hardening must not change MOS139 geometry.

Supported runtime: QGIS 4.0.2 Norrköping, using the committed memory-output
fixture matrix. File-output behavior remains governed by the existing plugin
output tests and does not alter the geometry lock.
